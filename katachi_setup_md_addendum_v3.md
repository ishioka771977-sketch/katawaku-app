## 【3D対応範囲｜JSONを書く前に必ず確認】（2026-07-02 追記）

### 描ける形（勾配は下記パラメータで渡す。生頂点・angle_deg・cut_noteで形を作らない）

| 形状 | 使うパラメータ |
|---|---|
| 擁壁の前面勾配 | `wall_slope`（または `thickness_top_mm` / `thickness_bottom_mm`） |
| 橋台竪壁のテーパ | `thickness_top_mm` / `thickness_bottom_mm` ＋ `taper: true` ＋ `taper_face`（`"back"`=背面のみ・既定／`"front"`／`"both"`=対称） |
| 橋台ウイング壁の縦断 | `height_start_mm` / `height_end_mm` |
| 床版のむくり（横断勾配） | `camber_mm`（2D断面＋3D誇張曲線に反映） |
| 床版の縦断勾配（橋軸方向） | `longitudinal_slope_percent`（%。正でA2側が上がる。3Dで床版・主桁・型枠一式が傾く） |
| 一様な線状ハンチ（地覆・壁高欄／BOX／床版主桁） | `haunch_w_mm`・`haunch_h_mm` 等の既存ハンチ項目（従来通り） |

### 描けない形（JSON化しない。「現状3D非対応」とひでさんに申告する）

- 横断勾配が橋軸方向に変化する真のねじれ面（むくり＋縦断の単純併用は可）
- 斜橋の斜角（平面的な斜角）
- 長さ方向に変化する非一様ハンチ（橋脚梁の支点ハンチ等）
- 他構造物との取り合い（地覆↔床版の打継ぎ、壁高欄↔地覆の目地一致、竪壁↔翼壁の接続角度）

### 禁止事項（描画に乗らない＝「作ったつもり」になる）

- `angle_deg`・`cut_note`・生の頂点座標で**形を作らない**。これらは3Dに反映されない。四隅の座標計算はアプリ（モジュール）側が行う。
- 未対応の形状は無理に近似JSONを出さず、**「現状3D非対応」と明示して申告**する。描けないものは書かない。

### 竪壁テーパ×リフトの書き方

`taper: true` ＋ `thickness_top_mm` / `thickness_bottom_mm` ＋ `taper_face` を指定。`lift_height_mm` / `lift_count` と併用可。各リフトの厚さは全体の高さから内挿されるため、リフトごとに厚さを書く必要はない。

### 床版の3D確認について

床版にも3D確認ビューが追加された（本体＋底鋼板＋主桁＋側型枠4面＋ハンチ帯＋むくり誇張曲線）。側型枠の折り畳み展開にも対応。
