> ボトルネックだった `PlaneGeometry(w,h)`（長方形固定）を「四隅自由」に一般化した。

## 1. 背景（現状の事実）

- 3D は「展開図を長方形の面に分けて折り畳む」方式（[3d.js](3d.js)）。
- 面の生成は `createFaceMesh(faceData, w, h)` → `THREE.PlaneGeometry(w, h)` の長方形＋キャンバステクスチャ。
- 各面は `folded`/`unfolded` の `{pos:[x,y,z], rot:[rx,ry,rz]}` を登録し、剛体として補間して折り畳む（[3d.js](3d.js)）。
- 旧状態では勾配（`wall_slope` 等）は **2D図・セパ計算表でのみ**使用され、3Dの面は長方形のまま固定Zに配置されていた。`angle_deg` は未使用、`cut_note` は描画未使用（注記止まり）。

## 2. スコープ

**解いた（今回）— 面そのものが非長方形（台形/平行四辺形）になるケース**
- テーパ壁の妻面（下底 `tBot` / 上底 `tTop`）。
- 縦断勾配で上辺と下辺の高さが違う面。
- 対称テーパ壁の前後面（面ごと傾く＋上端で内側へ）。

**解かない（別タスク）**
- 非一様ハンチ（長さ方向に変化）、角度付きカットの実線化、複数構造物の取り合い、曲線・ねじれ面。

> 「面ごと傾くだけ（長方形のまま回転）」は元々 `rot` で表現可能。今回必要だったのは
> **面そのものが非長方形**になるケース。

## 3. API（実装された形）

### 3.1 `createFaceMesh` を後方互換で拡張（[3d.js](3d.js)）

```js
// 既存呼び出し createFaceMesh(faceData, w, h) は完全に従来通り（長方形）。
// 第4引数 corners を渡すと任意四角形（台形）になる。
window.createFaceMesh = function(faceData, w, h, corners) {
  const tex = createFaceTexture(faceData, w, h);   // テクスチャは従来の「展開長方形」基準
  const geo = corners ? buildQuadGeometry(corners) : new THREE.PlaneGeometry(w, h);
  const mat = new THREE.MeshLambertMaterial({ map: tex, side: THREE.DoubleSide, transparent: true, opacity: 0.92 });
  return new THREE.Mesh(geo, mat);
};
```

- `corners`: 面ローカル平面上の4点 `[[x,y]×4]`（左下→右下→右上→左上）。省略時は長方形と一致＝後方互換。
- テクスチャは「展開した長方形」基準のまま。UV は四隅 → `(0,0)(1,0)(1,1)(0,1)` に固定マップ。
  台形では UV が引き伸ばされるが、型枠は「展開長で割付 → 空間では台形に置く」が物理的に正しい。

### 3.2 四隅ジオメトリ生成とヘルパ（[3d.js](3d.js)）

```js
buildQuadGeometry(corners)            // 四隅[LB,RB,RT,LT]→2三角形のBufferGeometry（z=0, UV固定）
quadTaper(wBot, wTop, h, lean)        // テーパ妻面の直角台形。lean='right'|'left'|'center'
quadTopSlope(w, hL, hR)               // 縦断勾配で天端が左右で高さ違いの台形（下辺水平）
```

> 既存コードも `THREE.Shape`/`ExtrudeGeometry` を使用済み（parapet 断面・box_culvert ハンチ）。
> 四隅 BufferGeometry はその延長で、依存追加なし。

## 4. 折り畳みアニメとの互換性

- フォールドは mesh.position/rotation を剛体補間するだけ。形状（台形）はジオメトリに焼き込むので、
  **フォールド処理は一切変更不要**。剛体の四角形が折り畳まれるだけ。
- 面ごと傾ける場合（前面傾斜・対称テーパ）は `folded.rot` に角度を入れる。傾き符号は
  実装時に THREE で実測して確定（例: 前面 `rot[+θ,π,0]` で上端が背面側へ）。

## 5. ハンチについて（訂正済み・重要）

当初は「parapet の45°近似ハンチを三角形化する」と想定していたが、実機検証で**訂正**した。

- 地覆/壁高欄・BOX のハンチは**延長方向に一様な線状の面取り**。
  `createFaceMesh(face, L, slant)` を実角度 `atan2(H, W)` で傾けた**長方形バンド**＝既に幾何学的に正しい
  （box は `ExtrudeGeometry` で押し出し＝同じく正しい）。
- 三角形ジオメトリが要るのは「長さ方向に変化する**非一様**ハンチ（隅角でテーパ等）」のみ。
- よって一様な線状ハンチは**変更しない**（"正規化"すると正しい描画を壊す）。

## 6. 各モジュールへの適用（実施内容）

- 変更は「非長方形にしたい面だけ」`createFaceMesh(..., corners)` に四隅を渡す。他面は引数省略で従来通り＝段階導入。
- **擁壁**: 妻面C/D=`quadTaper(tBot,tTop,wallH,…)`、前面A=傾斜（`rot[θ,π,0]`＋Zオフセット）。
- **橋台**: ウイング壁=`quadTopSlope(wLen, …)`、竪壁S-A/S-B（各リフト）=傾斜＋リフトごとのZオフセット。
- **床版**: 2D断面の天端を `camber_mm` ぶんのむくり（二次曲線）で描画。実寸不可視のため形状は誇張＋実値注記。

## 7. 設計原則（横展開時の鉄則）

> **JSON/md 側には「勾配パラメータ（`wall_slope`/`thickness_top_bottom`/`camber_mm` 等）」を書かせるだけ。
> 四隅の座標計算はモジュール内に閉じ込める。** 生の頂点座標を AI/JSON に書かせると破綻する。
> ＝ md 改善だけでは到達できず、本対応（四隅化）が前提。

## 8. 残課題と工数感

- 床版3Dビュー新規構築：他モジュール並みの build3D を camber 込みで新規開発（中〜大）。
- 非一様ハンチ・取り合い：三角形/多角形＋マルチ構造シーン（中〜大）。
- 既存の直方体モジュール（橋脚・基礎）：勾配要素なし＝今回対象外。

---

### まとめ
ボトルネックは `PlaneGeometry(w,h)` の長方形固定 1点だった。ここを「四隅自由」に一般化したことで、
折り畳みエンジン・テクスチャ・後方互換を壊さず、横断/縦断勾配・テーパを表現できるようになった
（擁壁・橋台・床版で実装・検証・本番反映済み）。一様な線状ハンチは元から正しく、変更不要。
JSON/md は勾配パラメータを渡すだけにし、四隅算出はモジュールに閉じ込めるのが要。
