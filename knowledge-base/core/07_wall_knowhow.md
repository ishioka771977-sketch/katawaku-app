# 擁壁型枠の知見

擁壁（retaining_wall）のJSON作成時に必要な専門知識。

---

## 擁壁の型枠の特徴

### 2回打設（底版→竪壁）

```
Phase 1: 底版
  ┌─────────────────────────┐
  │  底版 t=500             │  ← 控え杭方式（セパなし）
  └─────────────────────────┘

Phase 2: 竪壁
  ┌──────────────────┐
  │ 竪壁 H=3000      │  ← 対面セパ（前面B型/背面C型）
  │ 前面勾配1:0.05   │
  └──────────────────┘
```

### 表裏のセパ使い分け

| 面 | セパ | 理由 |
|-----|------|------|
| 前面（表面・仕上げ面） | B型（●）Pコン付き | 見える面。脱型後にPコン穴をモルタル充填 |
| 背面（裏面・埋戻し面） | C型（○）ナット式 | 埋め戻すので美観不問 |

---

## 勾配セパ長の自動計算（★差別化機能）

前面勾配がある場合、壁厚が上に行くほど薄くなる → セパ長が段ごとに異なる。

### 計算式

```
壁厚(h) = 底部壁厚 - (勾配 × セパ位置の高さ)

例: 底部300mm、勾配1:0.05、セパ位置の高さ450mm
  壁厚 = 300 - (0.05 × 450) = 300 - 22.5 = 277.5mm

B型セパ長 = 壁厚 + Pコン深さ(30mm) × 2 = 277.5 + 60 = 337.5mm → 338mm
C型セパ長 = 壁厚 + コンパネ厚(12mm) × 2 = 277.5 + 24 = 301.5mm → 302mm
```

**発注時は長さ別に集計が必要。** JSONの `separators.lengths_by_row_mm` で段ごとのセパ長を明記する。

---

## 底版の型枠

- 底版は幅が広い（2〜4m）が高さが低い（500〜1000mm）
- **セパは使わない** → 控え杭方式（木杭 or H鋼で外から押さえる）
- コンパネは横使い（900→550mmカット等）
- 天端は開放（コテ仕上げ）

---

## 打継ぎの処理

底版→竪壁の打継ぎは擁壁で最も重要な品質管理ポイント。

### 処理手順

1. 底版打設後、天端のレイタンスを除去（グリーンカット推奨）
2. チッピング処理（表面を粗くして付着力を確保）
3. 止水板（塩ビCC200×5）を打継ぎ面に設置
4. 竪壁の差し筋は底版打設前に配置済み
5. 湿潤養生後に竪壁打設

### JSONへの反映

```json
"construction_joint": {
  "base_to_wall": {
    "type": "horizontal",
    "treatment": "レイタンス除去・チッピング・湿潤",
    "waterstop": true,
    "waterstop_type": "PVC_CC200x5"
  }
}
```

## 水抜きパイプ

- VP75を2〜3m²に1箇所以上
- 前面側に下り勾配2%以上
- 背面にフィルター材（砕石＋透水シート）
- JSONでは `drain_pipe` オブジェクトと `faces[].drain_pipes` 配列で位置を指定

---

## 目地設計

- 延長10m超なら伸縮目地を入れる
- 間隔: @10〜15m
- 竪壁と底版を通しで目地を入れク（竪壁だけに入れてもダメ）
- **設計図に目地がない場合 → 技術提案の機会**
